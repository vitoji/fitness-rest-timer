# 微信小程序开发计划

## 1. 项目初始化

### 1.1 环境准备
- 安装微信开发者工具
- 注册微信小程序账号
- 创建小程序项目，选择"不使用云服务"模板

### 1.2 目录结构
```
wechat-miniprogram/
├── miniprogram/
│   ├── pages/
│   │   ├── index/         # 主页面（计时器和AI交互）
│   │   └── history/       # 训练记录页面
│   ├── components/        # 可复用组件
│   │   ├── timer/         # 计时器组件
│   │   ├── ai-chat/       # AI聊天组件
│   │   └── history-item/  # 训练记录项组件
│   ├── utils/             # 工具函数
│   │   ├── api.js         # API调用封装
│   │   ├── storage.js     # 本地存储封装
│   │   └── speech.js      # 语音功能封装
│   ├── app.js             # 小程序入口文件
│   ├── app.json           # 小程序配置文件
│   └── app.wxss           # 全局样式文件
├── cloudfunctions/        # 云函数（可选）
└── project.config.json    # 项目配置文件
```

## 2. 核心功能实现

### 2.1 计时器功能
- 实现与web版相同的倒计时功能
- 支持30s、60s、90s、120s、180s预设时长
- 实现开始/暂停/停止控制
- 结束时的视觉和振动提醒

### 2.2 AI交互功能
- 实现文本输入和发送功能
- 集成语音输入功能（使用微信小程序语音API）
- 实现语音合成功能（使用微信小程序语音API）
- 与现有后端API对接，保持三步骤处理逻辑：
  1. 意图识别
  2. 训练记录提取
  3. 生成回复

### 2.3 训练记录功能
- 实现训练记录的显示和管理
- 支持单条删除和批量清空
- 显示时间戳
- 本地存储训练记录

### 2.4 用户注册登录功能
- 实现基于微信账号的登录（微信小程序登录）
- 可选：实现手机号登录
- 实现用户信息存储和管理
- 实现不同用户的训练记录隔离

## 3. 技术栈选择

### 3.1 前端技术
- 微信小程序原生框架
- WXML + WXSS + JavaScript
- 微信小程序API（语音、存储、网络等）

### 3.2 后端技术
- 保持现有Express.js后端
- 部署在Vercel上
- 微信小程序通过HTTPS调用后端API

### 3.3 数据存储
- 本地存储：微信小程序本地存储
- 云端存储：后端数据库（可选，用于用户数据持久化）

## 4. 开发时间线

### 阶段一：项目初始化和基础功能（2天）
- 第1天：环境搭建，项目初始化，目录结构创建
- 第2天：实现计时器基本功能，页面布局

### 阶段二：核心功能实现（3天）
- 第3天：实现AI交互功能，API对接
- 第4天：实现训练记录功能
- 第5天：实现语音输入和输出功能

### 阶段三：用户系统和优化（2天）
- 第6天：实现用户注册登录功能
- 第7天：优化用户体验，测试和调试

## 7. 注意事项

### 7.1 微信小程序限制
- 小程序包大小限制（2MB）
- API调用频率限制
- 网络请求必须使用HTTPS
- 语音功能需要用户授权

### 7.2 性能优化
- 减少网络请求
- 优化本地存储使用
- 避免内存泄漏

### 7.3 用户体验
- 保持与web版一致的界面风格
- 优化移动端交互
- 提供清晰的用户引导

## 8. 风险评估

### 8.1 技术风险
- 微信小程序API与Web API的差异
- 语音功能在不同设备上的兼容性
- 后端API调用的稳定性

### 8.2 解决方案
- 充分测试语音功能在不同设备上的表现
- 实现API调用的错误处理和重试机制
- 提供降级方案，确保核心功能不受影响
