# 健身休息助手 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
健身休息助手是一款专注于健身组间休息的 Web 应用，帮助用户科学规划休息时间，提供 AI 互动和训练记录功能。

### 1.2 目标用户
- 健身爱好者
- 健身房训练人群
- 居家健身人群
- 需要科学规划休息时间的运动者

### 1.3 核心价值
- 提供精准的组间休息计时
- 支持多种休息时长选择
- AI 互动提供训练建议和鼓励
- 记录训练历史，方便回顾

## 2. 功能需求

### 2.1 倒计时功能
#### 2.1.1 时长选择
- 提供预设时长按钮：30秒、60秒、90秒、120秒、180秒
- 支持一键选择休息时长
- 在倒计时开始前可自由切换时长

#### 2.1.2 倒计时控制
- **开始按钮**：启动倒计时
- **暂停按钮**：暂停当前倒计时，可继续
- **停止按钮**：停止并重置倒计时
- **继续按钮**：从暂停状态继续倒计时

#### 2.1.3 倒计时显示
- 大号数字显示剩余时间
- 格式为 MM:SS（分:秒）
- 倒计时结束时有视觉闪烁效果

### 2.2 结束提醒
#### 2.2.1 弹窗提醒
- 倒计时结束后弹出全屏弹窗
- 弹窗显示"休息时间到！"的提醒
- 弹窗有动画效果，增强视觉体验
- 可通过点击按钮或背景关闭弹窗

#### 2.2.2 声音提醒
- 使用 Web Audio API 生成提示音
- 不依赖外部音频文件
- 播放三段提示音：两声短音，一声长音

#### 2.2.3 震动提醒
- 支持浏览器震动 API
- 设备支持时会有震动效果

#### 2.2.4 系统通知
- 请求浏览器通知权限
- 倒计时结束时发送系统通知
- 通知包含图标和鼓励文字

### 2.3 AI 互动功能
#### 2.3.1 自动 AI 消息
- 倒计时结束后自动发送 AI 消息
- 提供训练建议和鼓励话语
- 消息简短、贴合健身场景

#### 2.3.2 主动 AI 对话
- 用户可主动发送问题
- AI 提供健身相关建议
- 支持对话历史记录
- 最多保留 10 条对话历史

#### 2.3.3 加载状态
- AI 回复时显示加载指示器
- 包含旋转动画和"正在思考中..."文字
- 提升用户体验

### 2.4 训练记录功能
#### 2.4.1 记录保存
- 支持输入训练内容
- 保存到浏览器本地存储（localStorage）
- 可按回车或点击按钮保存

#### 2.4.2 历史查看
- 可查看所有训练历史
- 列表形式展示记录
- 支持展开/收起历史面板

## 3. 技术需求

### 3.1 技术栈
- **前端**：纯 HTML、CSS、JavaScript
- **后端**：Node.js + Express
- **部署平台**：Vercel
- **AI 服务**：MiniMax API

### 3.2 PWA 支持
- 支持添加到主屏幕
- 离线访问（Service Worker）
- 全屏显示模式
- 主题色设置

### 3.3 响应式设计
- 支持手机、平板、桌面设备
- 适配不同屏幕尺寸
- 移动端友好的触摸操作

## 4. 用户界面

### 4.1 主界面布局
- 顶部：应用标题
- 中部：倒计时显示区
- 中下部：时长选择按钮
- 下部：控制按钮区
- 底部：AI 互动区和训练记录区

### 4.2 颜色设计
- 主色调：绿色 (#4CAF50) - 代表健康和活力
- 开始按钮：蓝色 (#2196F3)
- 停止按钮：红色 (#f44336)
- 背景色：浅灰色 (#f0f0f0)
- 卡片背景：白色

### 4.3 交互设计
- 按钮悬停效果
- 点击反馈
- 平滑过渡动画
- 弹窗弹出动画

## 5. API 接口

### 5.1 聊天 API
- **端点**：`/api/chat`
- **方法**：POST
- **请求体**：
  ```json
  {
    "systemPrompt": "系统提示词",
    "userPrompt": "用户消息",
    "messageHistory": []
  }
  ```
- **响应**：
  ```json
  {
    "content": "AI 回复内容"
  }
  ```

### 5.2 AI 配置
- 使用 MiniMax-M2.7 模型
- 温度参数：1.0
- 支持上下文对话

## 6. 部署要求
